Better than markdown!? Using Racket & Pollen to create beautiful digital books

Short guide to using Racket the Pollen markup language
Author

JP Monteagudo

Published

Thursday, April 25, 2024

Isn’t markdown enough?

Markdown, unlike HTML, is easy to read and write in, which is why, in my opinion, it has become a commonly used markup language among non-programmers. We get rid of tags and clutter, substitute those for simple keyboard strokes, and suddenly our text becomes much less chaotic.

# Use `#` one hashtag for your first–level headings
Create Separate paragraphs using

line breaks.
## A second level heading using `##`?

1. Make your text **bold** or *italicized*
2. Create numbered
* And unnumbered lists
3. and add [links](https://www.jpmonteagudo.com), if you want to. 
#| It looks legible and anybody can do it

<ol>

  <li>Create numbered

      <li>And unnumbered lists</li>
    </ul>
  </li>
  <li>and add <a href="https://www.jpmonteagudo.com">links</a>, if you want to.</li>
</ol>
        The visual and aesthetic difference between markdown and HTML is glaringly obvious. HTML is meant to be a text editor and not a programming language, while markdown combines text-based editing with programming languages1.

1 Rmarkdown’s knitr can execute code in Python, SQL, Bash, Stan, JS, CSS to name a few languages

Enter Pollen

Pollen is a publishing system built on top of Scribble and Racket2. This markup language is typically used to create digital books, PDFs and other types of files as its output.

2 Racket is a programming language based on Lisp
